Приложение 1

ВАРИАНТЫ ИНДИВИДУАЛЬНЫХ ЗАДАНИЙ

 

В первом абзаце каждого задания содержится информация о предметной области, для которой должна быть разработана модель реляционной БД и приведены ориентировочные таблицы, состав которых может быть изменен студентом. Жирным шрифтом выделена таблица, отражающая основную функцию, которая будет реализована с помощью создаваемой БД (как правило, это зависимая таблица, часть полей которой является внешними ключами на родительские таблицы-справочники). Общее количество полей в базе – не менее 20, общее количество записей – не менее 30. Эта часть задания используется в лабораторных работах №1, 2, 8.

Второй абзац содержит перечень запросов, которые необходимо создать и выполнить в СУБД MSAccess, а результат представить в виде отчета.  Эта часть задания используется в лабораторных работах № 3, 4.
